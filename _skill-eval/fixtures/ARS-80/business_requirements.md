# Бизнес-требования: Панель активности пользователей в админке «Арсенал»

> **Статус готовности:** Готово к оценке

> **Открытые вопросы (требуют уточнения перед оценкой):**

## 1. Введение и контекст
### 1.0. Тип изменения
Доработка существующего — в веб-админку «Арсенал» добавляется несколько связанных элементов для наблюдения за активностью пользователей.
### 1.1. Описание проблемы (Problem Statement)
Администраторы не видят ни текущую, ни историческую активность пользователей: нельзя быстро оценить нагрузку, посмотреть динамику за период и забрать сводку для отчётности. Из-за этого регламентные работы планируются вслепую.
### 1.2. Конечные потребители (End Users)
Администраторы системы «Арсенал» (внутренние сотрудники эксплуатации).
### 1.3. Заказчик и стейкхолдеры (Stakeholder / Sponsor)
Заказчик — руководитель отдела эксплуатации. Эксперт для консультации — тимлид команды «Арсенал-Core».

## 2. Бизнес-требования (цели и ценность)
### 2.1. Бизнес-цель
Я как администратор должен видеть активность пользователей — сейчас, в динамике и в виде выгружаемой сводки, — чтобы планировать безопасные окна работ и отчитываться по нагрузке.
### 2.2. Ценность для бизнеса
Меньше прерванных сессий во время регламентных работ и прозрачная отчётность по нагрузке для аудита и планирования.
### 2.3. Бизнес-риски и меры по снижению (Mitigation)
| Риск | Вероятность / влияние | Меры по снижению |
|------|-----------------------|------------------|
| Неточные данные активности вводят в заблуждение | Средняя / среднее | При недоступности источника показывать «—»/пустой график, не выдавать устаревшее за актуальное |
| Выгрузка доступна не тем ролям | Низкая / среднее | Панель и выгрузка — только у администраторов |

## 3. Объём проекта (Scope)
### 3.1. Входит в проект (In-scope)
- Счётчик активных пользователей в шапке админки, обновляемый без перезагрузки.
- Модальное окно с графиком активности пользователей за выбранный период (час / день / неделя), открывается по клику на счётчик.
- Кнопка в модальном окне для выгрузки сводки активности за выбранный период в файл.
### 3.2. Не входит в проект (Out-of-scope)
- Список конкретных пользователей онлайн (только агрегированные числа).
- Автоматическая рассылка отчётов.
- Прогнозирование будущей активности.

### 3.3. Архитектурные изменения и интеграции
Все три элемента опираются на одни и те же данные об активности пользователей (текущее число активных и агрегированная активность за период). Админка «Арсенал» начинает потреблять эти данные. Точные механизмы получения и хранения — на этапе тех.спеки.

## 4. Функциональные требования
### 4.1. Описание функционала / бизнес-процесса
Администратор видит в шапке живой счётчик, кликом открывает модалку с графиком активности за период и может выгрузить сводку за тот же период.
- **FR-1.** Система отображает в шапке админки число активных пользователей.
- **FR-2.** Значение счётчика обновляется автоматически, без перезагрузки страницы.
- **FR-3.** По клику на счётчик открывается модальное окно с графиком активности пользователей за выбранный период (час / день / неделя).
- **FR-4.** В модальном окне администратор может переключать период графика.
- **FR-5.** В модальном окне есть кнопка выгрузки сводки активности за выбранный период в файл.
### 4.2. Критерии приёмки (Acceptance Criteria)
- **[FR-1]** Дано открытую админку, когда страница загрузилась, тогда в шапке видно число активных пользователей.
- **[FR-2]** Дано открытую админку, когда число активных пользователей изменилось, тогда счётчик обновляется без перезагрузки в пределах минуты.
- **[FR-3]** Дано счётчик в шапке, когда администратор кликает по нему, тогда открывается модальное окно с графиком активности за период по умолчанию.
- **[FR-4]** Дано открытую модалку, когда администратор выбирает другой период, тогда график перестраивается под выбранный период.
- **[FR-5]** Дано открытую модалку с выбранным периодом, когда администратор нажимает выгрузку, тогда он получает файл со сводкой активности за этот период.

## 5. Ограничения и атрибуты качества
### 5.1. Приоритет
Medium — улучшает эксплуатацию, но не блокирует основной бизнес.
### 5.2. Сроки и обоснование
До конца квартала — привязано к плану регламентных работ.
### 5.3. Ролевая модель
| Роль | Права / доступные действия |
|------|----------------------------|
| Администратор | Видит счётчик, график активности и выгрузку сводки |
| Оператор | Панель активности не отображается |
